# Day3 练习题：事务、索引与查询优化

这一天专门练事务可靠性、索引和基础 SQL 分析。

## 题目 1：创建账户表

创建 `accounts` 表，字段包括：

- `id`
- `name`
- `balance`

要求：

- `id` 是自增主键。
- `balance` 使用 `DECIMAL`。
- 插入至少 2 个账户。

## 题目 2：成功转账

使用事务完成一次转账：

- A 扣 100。
- B 加 100。

要求：

- 使用 `START TRANSACTION`。
- 使用 `COMMIT`。
- 转账前后查询余额。

## 题目 3：回滚转账

模拟一次错误转账，然后执行 `ROLLBACK`。

要求：

- 事务中先扣款。
- 不执行加款。
- 执行 `ROLLBACK`。
- 查询余额是否恢复。

## 题目 4：余额不足判断

写一条扣款 SQL：

```sql
UPDATE accounts
SET balance = balance - 100
WHERE id = 1 AND balance >= 100;
```

要求：

- 观察余额不足时影响行数。
- 说明为什么 `balance >= 100` 很重要。

## 题目 5：创建日志表

创建 `access_logs` 表，字段包括：

- `id`
- `user_id`
- `url`
- `created_at`

要求：

- 插入至少 10 条日志。
- 不同用户要有不同日志。

## 题目 6：创建普通索引

给 `access_logs.user_id` 创建索引。

要求：

- 使用 `CREATE INDEX`。
- 使用 `SHOW INDEX FROM access_logs` 查看索引。

## 题目 7：使用 EXPLAIN

对下面查询使用 `EXPLAIN`：

```sql
SELECT * FROM access_logs WHERE user_id = 1;
```

要求：

- 观察 `key`。
- 观察 `rows`。
- 用自己的话解释结果。

## 题目 8：创建联合索引

给 `access_logs` 创建联合索引：

```sql
CREATE INDEX idx_logs_user_created ON access_logs(user_id, created_at);
```

要求：

- 查询某个用户最近的访问记录。
- 使用 `EXPLAIN` 观察是否使用索引。

## 题目 9：索引代价

请解释为什么索引不是越多越好。

要求：

- 至少说明空间成本。
- 至少说明写入成本。

## 题目 10：综合小练习

设计一个简单订单支付场景：

- 用户账户表
- 订单表
- 支付时扣余额
- 更新订单状态

要求：

- 扣余额和改订单状态必须在同一个事务里。
- 给 `orders.user_id` 建索引。
- 用 `EXPLAIN` 分析按用户查询订单的 SQL。

## 检查标准

如果你做完了，至少应该能清楚解释：

- 事务为什么能保证转账安全。
- `COMMIT` 和 `ROLLBACK` 的区别。
- ACID 四个特性的大概含义。
- 索引适合加在哪些字段上。
- `EXPLAIN` 里的 `key` 和 `rows` 大概代表什么。
