# Day2 练习题：表关系、JOIN 与约束

这一天专门练多表关系、JOIN 查询和约束。

## 题目 1：设计用户和文章表

创建两张表：

- `users`
- `posts`

要求：

- 一个用户可以有多篇文章。
- `posts.user_id` 表示文章作者。
- 用户邮箱使用 `UNIQUE`。
- 文章标题不能为空。

## 题目 2：插入测试数据

插入至少：

- 3 个用户
- 5 篇文章

要求：

- 至少有一个用户没有文章。
- 至少有一个用户有多篇文章。

## 题目 3：查询文章作者

查询文章列表，并显示作者名称。

要求：

- 使用 `INNER JOIN`。
- 查询字段包含文章标题、作者名称、发布时间。

## 题目 4：查询所有用户和文章数量

查询每个用户发布了多少篇文章。

要求：

- 使用 `LEFT JOIN`。
- 使用 `COUNT`。
- 没有文章的用户也要显示。

## 题目 5：查询没有文章的用户

找出没有发布文章的用户。

要求：

- 使用 `LEFT JOIN`。
- 使用 `WHERE posts.id IS NULL`。

## 题目 6：设计学生选课表

创建三张表：

- `students`
- `courses`
- `student_courses`

要求：

- 使用中间表表达多对多关系。
- 中间表包含 `student_id` 和 `course_id`。

## 题目 7：查询学生课程

查询每个学生选择了哪些课程。

要求：

- 至少使用 2 次 JOIN。
- 展示学生姓名和课程名称。

## 题目 8：分组统计课程人数

统计每门课程有多少学生选择。

要求：

- 使用 `GROUP BY`。
- 使用 `COUNT`。

## 题目 9：唯一约束测试

尝试插入两个相同邮箱的用户。

要求：

- 观察 MySQL 报错。
- 说明 `UNIQUE` 的作用。

## 题目 10：综合小练习

设计一个简单博客系统表结构：

- 用户
- 文章
- 评论

要求：

- 用户和文章是一对多。
- 文章和评论是一对多。
- 评论属于一个用户。
- 写出查询“文章标题、作者、评论数量”的 SQL。

## 检查标准

如果你做完了，至少应该能清楚解释：

- 一对多和多对多分别怎么建表。
- 为什么多对多需要中间表。
- `INNER JOIN` 和 `LEFT JOIN` 的区别。
- 外键、唯一约束、非空约束分别解决什么问题。
- `GROUP BY` 为什么常和聚合函数一起使用。
