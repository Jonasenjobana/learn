# Day1 练习答案：数据库基础、建表与 CRUD

这份答案对应 [练习题](./练习题.md)。  
建议先自己写一遍，再对照这里检查 SQL 写法。

## 题目 1：创建数据库

```sql
CREATE DATABASE IF NOT EXISTS learn_shop DEFAULT CHARACTER SET utf8mb4;
USE learn_shop;
```

说明：

- `IF NOT EXISTS` 可以避免数据库已存在时报错。
- `utf8mb4` 能更好支持中文和 emoji。

## 题目 2：创建商品表

```sql
DROP TABLE IF EXISTS products;

CREATE TABLE products (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100) NOT NULL,
  price DECIMAL(10, 2) NOT NULL DEFAULT 0.00,
  stock INT NOT NULL DEFAULT 0,
  description TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

说明：

- `DECIMAL(10, 2)` 适合保存金额，最多 10 位数字，其中 2 位小数。
- `description` 可以为空，所以不加 `NOT NULL`。
- `created_at` 不传值时会自动使用当前时间。

## 题目 3：插入数据

```sql
INSERT INTO products (name, price, stock, description)
VALUES
  ('机械键盘', 199.00, 12, '适合办公和游戏'),
  ('无线鼠标', 89.90, 30, '轻量化无线鼠标'),
  ('显示器支架', 159.00, 8, '可升降桌面支架'),
  ('Type-C 数据线', 29.90, 100, '1.5 米快充线'),
  ('蓝牙耳机', 299.00, 0, '主动降噪入耳式耳机');
```

说明：

- `机械键盘`、`显示器支架`、`蓝牙耳机` 的价格都大于 100。
- `蓝牙耳机` 的库存为 0。

## 题目 4：基础查询

查询所有商品：

```sql
SELECT * FROM products;
```

只查询指定字段：

```sql
SELECT name, price, stock
FROM products;
```

说明：

- 学习阶段可以用 `SELECT *`。
- 真实接口里更推荐明确写出需要的字段，避免多查无用数据。

## 题目 5：条件查询

```sql
SELECT name, price
FROM products
WHERE price > 100;
```

说明：

- `WHERE` 用于筛选行。
- 这里只返回商品名和价格。

## 题目 6：排序和限制数量

```sql
SELECT id, name, price, stock
FROM products
ORDER BY price DESC
LIMIT 3;
```

说明：

- `ORDER BY price DESC` 表示按价格从高到低排序。
- `LIMIT 3` 表示只取前 3 条。

## 题目 7：更新库存

先查询目标商品：

```sql
SELECT *
FROM products
WHERE id = 1;
```

更新库存：

```sql
UPDATE products
SET stock = 20
WHERE id = 1;
```

更新后确认：

```sql
SELECT *
FROM products
WHERE id = 1;
```

说明：

- `UPDATE` 必须带准确的 `WHERE`。
- 如果不带 `WHERE`，整张表的库存都会被改掉。

## 题目 8：删除商品

先确认要删除的数据：

```sql
SELECT *
FROM products
WHERE id = 5;
```

删除指定商品：

```sql
DELETE FROM products
WHERE id = 5;
```

删除后确认：

```sql
SELECT *
FROM products;
```

说明：

- `DELETE FROM products WHERE id = 5` 只删除指定商品。
- `DELETE FROM products` 会删除整张表所有数据，要非常小心。

## 题目 9：空值和默认值

```sql
INSERT INTO products (name, price, stock)
VALUES ('笔记本支架', 69.00, 15);

SELECT *
FROM products
WHERE name = '笔记本支架';
```

观察结果：

- `description` 没有传值，结果通常是 `NULL`。
- `created_at` 没有传值，会自动生成当前时间。

## 题目 10：综合小练习

下面是一份完整答案，可以从头执行：

```sql
CREATE DATABASE IF NOT EXISTS learn_shop DEFAULT CHARACTER SET utf8mb4;
USE learn_shop;

DROP TABLE IF EXISTS products;

CREATE TABLE products (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100) NOT NULL,
  price DECIMAL(10, 2) NOT NULL DEFAULT 0.00,
  stock INT NOT NULL DEFAULT 0,
  description TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

INSERT INTO products (name, price, stock, description)
VALUES
  ('机械键盘', 199.00, 12, '适合办公和游戏'),
  ('无线鼠标', 89.90, 30, '轻量化无线鼠标'),
  ('显示器支架', 159.00, 8, '可升降桌面支架'),
  ('Type-C 数据线', 29.90, 100, '1.5 米快充线'),
  ('蓝牙耳机', 299.00, 0, '主动降噪入耳式耳机');

SELECT * FROM products;

SELECT name, price
FROM products
WHERE price > 100;

SELECT id, name, price, stock
FROM products
ORDER BY price DESC
LIMIT 3;

UPDATE products
SET stock = 20
WHERE id = 1;

SELECT * FROM products WHERE id = 1;

DELETE FROM products
WHERE id = 5;

SELECT * FROM products;
```

## 对照检查

你应该能解释：

- 数据库像项目的数据空间，表像一种数据集合，字段是列，行是一条记录。
- `VARCHAR` 适合短文本，`TEXT` 适合长文本，`DECIMAL` 适合金额，`DATETIME` 适合时间。
- `UPDATE` 和 `DELETE` 如果没有 `WHERE`，可能影响整张表。
- `ORDER BY` 用于排序，`LIMIT` 用于限制返回数量。
