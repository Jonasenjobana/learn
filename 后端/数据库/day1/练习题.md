# Day1 练习题：数据库基础、建表与 CRUD

这一天专门练 MySQL 最基础的表结构和增删改查。

## 题目 1：创建数据库

创建一个数据库：

```sql
learn_shop
```

要求：

- 字符集使用 `utf8mb4`。
- 创建后切换到该数据库。

## 题目 2：创建商品表

创建 `products` 表，字段包括：

- `id`
- `name`
- `price`
- `stock`
- `description`
- `created_at`

要求：

- `id` 是自增主键。
- `name` 不能为空。
- `price` 使用 `DECIMAL`。
- `created_at` 默认当前时间。

## 题目 3：插入数据

向 `products` 表插入至少 5 条数据。

要求：

- 至少有 2 条商品价格大于 100。
- 至少有 1 条商品库存为 0。

## 题目 4：基础查询

查询所有商品。

要求：

- 使用 `SELECT *`。
- 再写一条只查询 `name`、`price`、`stock` 的 SQL。

## 题目 5：条件查询

查询价格大于 100 的商品。

要求：

- 使用 `WHERE`。
- 查询结果只展示商品名和价格。

## 题目 6：排序和限制数量

查询价格最高的 3 个商品。

要求：

- 使用 `ORDER BY`。
- 使用 `LIMIT`。

## 题目 7：更新库存

把某个商品的库存改成 20。

要求：

- 必须使用 `WHERE id = ?`。
- 更新后重新查询该商品。

## 题目 8：删除商品

删除一条指定商品。

要求：

- 必须使用 `WHERE`。
- 删除前先用 `SELECT` 确认目标商品。

## 题目 9：空值和默认值

插入一条没有 `description` 的商品。

要求：

- 观察 `description` 的值。
- 观察 `created_at` 是否自动生成。

## 题目 10：综合小练习

做一个简单商品表练习，完成：

- 建库
- 建表
- 插入数据
- 条件查询
- 排序
- 更新
- 删除

## 检查标准

如果你做完了，至少应该能清楚解释：

- 数据库、表、字段、行分别是什么。
- `VARCHAR`、`TEXT`、`DECIMAL`、`DATETIME` 的常见用途。
- 为什么 `UPDATE` 和 `DELETE` 必须小心 `WHERE`。
- `ORDER BY` 和 `LIMIT` 分别做什么。
