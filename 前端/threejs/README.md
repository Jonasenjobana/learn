## 阶段总结

### Day1 总结：最小三维场景与动画循环

Day1 的目标是先把 Three.js 项目跑起来，不急着做复杂模型或高级特效。你会从一个旋转立方体开始，理解三维页面最核心的搭建顺序。

重点内容：

- 用 Vite 启动一个轻量前端示例。
- 创建 `Scene` 作为三维世界容器。
- 使用 `PerspectiveCamera` 观察场景。
- 使用 `WebGLRenderer` 把画面渲染到页面。
- 通过 `BoxGeometry`、`MeshStandardMaterial` 和 `Mesh` 创建一个立方体。
- 添加基础灯光，让材质有明暗变化。
- 使用动画循环让物体持续旋转。
- 监听窗口尺寸变化，避免画面拉伸。

适合目标：

- 能执行 `npm install` 后通过 `npm run dev` 启动示例。
- 能解释 Three.js 中场景、相机、渲染器的职责。
- 能修改立方体尺寸、颜色、旋转速度和相机位置。

## 学习目标

学完当前阶段后，你应该能：

- 独立搭建一个最小 Three.js 页面。
- 看懂 Three.js 示例项目的基础目录结构。
- 理解 3D 坐标、相机视角和渲染循环的关系。
- 在已有前端项目中判断 Three.js 代码应该放在哪里。

## 常见提醒

- Three.js 依赖浏览器 WebGL 能力，示例应通过本地开发服务器访问，不建议直接双击 HTML 文件。
- 不要一开始就追求复杂模型，先掌握场景、相机、渲染器和动画循环。
- 画面空白时，优先检查相机位置、物体位置、渲染器是否挂载到页面，以及浏览器控制台报错。
- 需要安装依赖时，本仓库沿用 npm，进入对应 day 目录后执行 `npm install`。
