# webpack个人总结
## 构建依赖
webpack通过entry入口开始构建依赖关系。
- js文件夹中引入像css、图片资源等，会将其当作依赖模块处理。如果有style-loader会将css内容变为style标签嵌入到html中；如果被 import 的图片会走 webpack 的 asset 模块处理：小于 maxSize（如 4KB）的转 Base64 内联，大于的则输出为独立文件并加上 contenthash。
> webpack只认识js/json因此很多非此资源需要用到module.rules里的loader。
- webpack配置中，很多通过正则表达式去匹配。webpack内部也把代码解析为AST，保证只有import 部分的路径符合正则才会去执行
## 加载器
### css
1. mini-css-extract-plugin和style-loader
这两个通常是互斥使用的，mini-css-extract-plugin会输出css文件，而style-loader是直接生成style标签嵌入到html。
> 开发者用style-loader热更新加载快（直接替换style) 

|  | mini-css-extract-plugin | style-loader |
| --- | --- | --- |
| 开发环境 | ❌ | ✅  |
| 热更新速度 | 慢 | 极快 |
| 首屏 | 并行 | 闪白（js执行后才注入） |
| 文件大小 | 分离 | 包体积更大 |
