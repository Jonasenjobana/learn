# Day3 练习题：Babel 与代码分割

这一天重点练 webpack 和 Babel 的配合，以及动态导入带来的代码分割。

## 题目 1：解释分工

请用自己的话解释：

- webpack 负责什么。
- Babel 负责什么。
- 为什么 webpack 需要通过 `babel-loader` 使用 Babel。

要求：

- 不要只背概念。
- 至少结合一个例子说明。

## 题目 2：阅读 Babel 配置

打开 `babel.config.js`，解释下面配置的含义：

```js
targets: '> 0.5%, last 2 versions, not dead'
```

要求：

- 说明它影响什么。
- 说明目标浏览器越旧会发生什么。

## 题目 3：修改现代语法

在 `src/index.js` 中增加至少 2 种现代 JS 语法，例如：

- 可选链 `?.`
- 空值合并 `??`
- 模板字符串
- 箭头函数
- 解构赋值

要求：

- 页面仍然能正常运行。
- 执行 `npm run build` 后观察输出代码。

## 题目 4：新增一个异步模块

在 `src/modules` 下新增一个文件，例如 `message.js`。

要求：

- 导出一个函数。
- 在 `src/index.js` 中用 `import()` 动态导入。
- 点击按钮后再加载并执行它。

## 题目 5：观察 chunk

执行：

```bash
npm run build
```

然后查看 `dist/js`。

要求：

- 找到主入口文件。
- 找到 runtime 文件。
- 找到异步 chunk 文件。
- 说明每个文件大概负责什么。

## 题目 6：动态导入改普通导入

把 `import()` 改成普通顶部 `import`。

要求：

- 再次执行构建。
- 对比 `dist/js` 文件数量。
- 说明为什么文件数量发生变化。

## 题目 7：理解 webpackChunkName

找到代码中的：

```js
/* webpackChunkName: "stats" */
```

要求：

- 修改 chunk 名称。
- 重新构建。
- 观察输出文件名变化。

## 题目 8：关闭 runtimeChunk

把配置中的：

```js
runtimeChunk: 'single'
```

临时注释掉。

要求：

- 重新构建。
- 对比输出文件。
- 说明 runtime 单独拆出来的作用。

## 题目 9：source map 对比

修改 `devtool`：

```js
devtool: isProd ? false : 'eval-cheap-module-source-map'
```

要求：

- 对比生产构建是否生成 `.map` 文件。
- 说明 source map 的作用。
- 思考生产环境是否应该公开 source map。

## 题目 10：综合小练习

完成一个页面：

- 首屏只显示标题、说明和按钮。
- 点击按钮后动态加载一个模块。
- 异步模块负责计算一组分数的最高分、最低分、平均分。
- 页面展示计算结果。

完成后你应该能解释：

- 为什么异步模块不会进入首屏主包。
- Babel 在这个项目里处理了哪些语法。
- `splitChunks` 和 `runtimeChunk` 分别解决什么问题。

## 检查标准

如果你做完了，至少应该能清楚解释：

- webpack 和 Babel 的职责区别。
- `babel-loader` 的作用。
- `@babel/preset-env` 和 `targets` 的关系。
- 普通导入和动态导入对打包结果的影响。
- 为什么真实项目需要代码分割。
