# Day2 练习题：webpack 静态资源与生产构建

这一天专门练 webpack 5 的静态资源处理能力，不再重复最基础的入口和出口配置。

## 题目 1：区分资源目录

请解释下面两类资源分别应该放在哪里：

- 页面组件中会被 `import` 的图片
- `favicon.ico` 或固定名称的第三方配置文件

要求：

- 说明 `src/assets` 和 `public` 的区别。
- 至少举 2 个例子。

## 题目 2：JS 中引入图片

在 `src/assets` 中放一张图片或 SVG，然后在 `src/index.js` 中引入并插入页面。

要求：

- 使用 `import imageUrl from './assets/xxx'`
- 创建 `img` 标签
- 设置 `img.src`

## 题目 3：CSS 中使用背景图

在 `src/style.css` 中给某个元素加背景图。

要求：

- 使用 `background-image: url('./assets/xxx')`
- 构建后确认图片被输出到 `dist/images`

## 题目 4：理解 `asset/resource`

把图片规则改成：

```js
type: 'asset/resource'
```

然后执行构建，观察输出结果。

要求：

- 说明它和 `asset/inline` 的区别。
- 说明为什么大图片更适合 `asset/resource`。

## 题目 5：理解 `asset/inline`

把一个很小的 SVG 图标配置成：

```js
type: 'asset/inline'
```

要求：

- 构建后观察 JS 文件内容。
- 说明 Data URL 的优点和缺点。

## 题目 6：理解 `asset`

使用下面配置：

```js
type: 'asset',
parser: {
  dataUrlCondition: {
    maxSize: 4 * 1024,
  },
}
```

要求：

- 修改 `maxSize` 的值。
- 观察资源是被内联还是被输出成文件。
- 用自己的话解释 webpack 的判断规则。

## 题目 7：导入文本内容

新建一个 `.txt` 文件，通过 `asset/source` 导入并显示在页面上。

要求：

- 配置 `test: /\.txt$/i`
- 使用 `type: 'asset/source'`
- 页面上能看到文本内容。

## 题目 8：复制 public 静态目录

在 `public/static` 下新增一个 JSON 文件，例如：

```json
{
  "name": "webpack day2",
  "version": "1.0.0"
}
```

要求：

- 执行 `npm run build`
- 确认文件被复制到 `dist/static`
- 说明为什么这类文件不一定要放进 `src/assets`

## 题目 9：生产环境 CSS

执行：

```bash
npm run build
```

观察 `dist/css` 目录。

要求：

- 找到构建后的 CSS 文件。
- 说明为什么生产环境要抽离 CSS。
- 说明 `contenthash` 的作用。

## 题目 10：综合小练习

做一个简单页面，要求包含：

- 一个标题
- 一段从 `.txt` 导入的介绍文字
- 一张通过 JS 插入的图片
- 一个通过 CSS 背景图展示的区域
- 一个从 `public/static` 复制出来的 JSON 文件

完成后你应该能解释：

- 哪些资源参与 webpack 构建。
- 哪些资源只是原样复制。
- 为什么构建后的文件名会带 hash。
- 开发环境和生产环境的样式处理有什么不同。

## 检查标准

如果你做完了，至少应该能清楚解释：

- `asset/resource`、`asset/inline`、`asset/source`、`asset` 的区别。
- `src/assets` 和 `public` 的区别。
- `contenthash` 为什么对缓存有帮助。
- 为什么 webpack 5 基础静态资源场景不再需要 `file-loader` 和 `url-loader`。
