# Day2 练习题：Dockerfile 与镜像构建

这一天专门练 Dockerfile 和自定义镜像构建。

## 题目 1：阅读 Dockerfile

打开 [Dockerfile](./Dockerfile)。

要求：

- 说明 `FROM` 的作用。
- 说明 `WORKDIR` 的作用。
- 说明 `CMD` 是什么时候执行的。

## 题目 2：构建镜像

执行：

```bash
docker build -t docker-day2-node .
```

要求：

- 构建成功。
- 使用 `docker images` 找到镜像。

## 题目 3：运行容器

执行：

```bash
docker run -d --name docker-day2-app -p 3000:3000 docker-day2-node
```

要求：

- 浏览器访问 `http://localhost:3000`。
- 能看到返回内容。

## 题目 4：查看日志

执行：

```bash
docker logs docker-day2-app
```

要求：

- 找到服务启动日志。

## 题目 5：修改业务代码

修改 `app/server.js` 中的响应内容。

要求：

- 重新 build。
- 重新 run。
- 浏览器看到新内容。

## 题目 6：理解构建缓存

连续执行两次：

```bash
docker build -t docker-day2-node .
```

要求：

- 观察第二次构建是否更快。
- 说明缓存的作用。

## 题目 7：使用 .dockerignore

打开 [.dockerignore](./.dockerignore)。

要求：

- 说明为什么不应该把 `node_modules` 放进构建上下文。
- 添加一个你认为应该忽略的文件类型。

## 题目 8：修改端口映射

把运行命令改成：

```bash
docker run -d --name docker-day2-app -p 3001:3000 docker-day2-node
```

要求：

- 浏览器访问 `http://localhost:3001`。
- 说明 `3001:3000` 的含义。

## 题目 9：停止并删除容器

执行：

```bash
docker stop docker-day2-app
docker rm docker-day2-app
```

要求：

- 删除前用 `docker ps -a` 确认容器名。

## 题目 10：综合小练习

完整完成：

- 写 Dockerfile。
- 构建镜像。
- 运行容器。
- 访问服务。
- 修改代码。
- 重新构建运行。
- 停止删除容器。

## 检查标准

如果你做完了，至少应该能清楚解释：

- Dockerfile 和镜像的关系。
- 构建上下文是什么。
- `.dockerignore` 的作用。
- `RUN` 和 `CMD` 的区别。
- 为什么 Node 项目通常先复制 `package.json` 再复制源码。
