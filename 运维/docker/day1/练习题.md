# Day1 练习题：镜像、容器与常用命令

这一天专门练 Docker 最基础的运行命令。

## 题目 1：查看 Docker 版本

执行：

```bash
docker version
```

要求：

- 能看到 Client 和 Server 信息。
- 如果 Server 看不到，说明 Docker 服务可能没有启动。

## 题目 2：拉取 nginx 镜像

执行：

```bash
docker pull nginx
```

要求：

- 拉取成功后执行 `docker images`。
- 找到 `nginx` 镜像。

## 题目 3：运行 nginx 容器

启动一个容器：

```bash
docker run -d --name web -p 8080:80 nginx
```

要求：

- 容器名叫 `web`。
- 浏览器访问 `http://localhost:8080`。

## 题目 4：查看容器状态

执行：

```bash
docker ps
docker ps -a
```

要求：

- 说明两个命令的区别。
- 找到 `web` 容器。

## 题目 5：查看日志

执行：

```bash
docker logs web
```

要求：

- 浏览器访问一次 nginx。
- 再查看日志，观察是否出现访问记录。

## 题目 6：进入容器

执行：

```bash
docker exec -it web sh
```

要求：

- 进入容器后执行 `ls`。
- 执行 `exit` 退出容器。

## 题目 7：停止容器

执行：

```bash
docker stop web
```

要求：

- 再执行 `docker ps`。
- 确认 `web` 不在运行列表中。

## 题目 8：删除容器

执行：

```bash
docker rm web
```

要求：

- 删除前确认容器已经停止。
- 执行 `docker ps -a` 确认容器不存在。

## 题目 9：换端口运行

重新运行 nginx：

```bash
docker run -d --name web2 -p 8081:80 nginx
```

要求：

- 浏览器访问 `http://localhost:8081`。
- 说明 `8081:80` 的含义。

## 题目 10：综合小练习

完成一整套流程：

- 拉取镜像。
- 启动容器。
- 浏览器访问。
- 查看日志。
- 进入容器。
- 停止容器。
- 删除容器。

## 检查标准

如果你做完了，至少应该能清楚解释：

- 镜像和容器的区别。
- `-d`、`--name`、`-p` 的作用。
- `docker ps` 和 `docker ps -a` 的区别。
- 为什么删除容器前通常要先停止。
