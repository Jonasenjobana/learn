# Day3 练习题：Docker Compose 与多容器应用

这一天专门练 Compose 多容器管理。

## 题目 1：阅读 compose.yml

打开 [compose.yml](./compose.yml)。

要求：

- 找到 `app` 服务。
- 找到 `mysql` 服务。
- 找到 `redis` 服务。
- 说明每个服务大概做什么。

## 题目 2：启动服务

执行：

```bash
docker compose up -d
```

要求：

- 启动成功。
- 使用 `docker compose ps` 查看服务状态。

## 题目 3：访问 app

浏览器访问：

```text
http://localhost:3000
```

要求：

- 能看到 JSON 响应。
- 响应中包含 MySQL host 和 Redis host。

## 题目 4：查看日志

执行：

```bash
docker compose logs app
docker compose logs mysql
```

要求：

- 找到 app 启动日志。
- 找到 MySQL 初始化日志。

## 题目 5：理解服务名通信

观察 app 的环境变量：

```yaml
MYSQL_HOST: mysql
REDIS_HOST: redis
```

要求：

- 说明为什么不是 `localhost`。
- 说明 Compose 服务名的作用。

## 题目 6：修改端口

把 app 端口映射改成：

```yaml
ports:
  - "3001:3000"
```

要求：

- 重新启动服务。
- 浏览器访问 `http://localhost:3001`。

## 题目 7：进入容器

执行：

```bash
docker compose exec app sh
```

要求：

- 进入 app 容器。
- 执行 `env` 查看环境变量。
- 执行 `exit` 退出。

## 题目 8：进入 MySQL

执行：

```bash
docker compose exec mysql mysql -u root -p
```

要求：

- 输入 root 密码。
- 执行 `SHOW DATABASES;`。

## 题目 9：理解 volume

停止服务：

```bash
docker compose down
```

再启动：

```bash
docker compose up -d
```

要求：

- 观察 MySQL 数据是否保留。
- 说明 volume 的作用。

## 题目 10：综合小练习

完成一整套 Compose 流程：

- 启动多服务。
- 查看服务状态。
- 访问 app。
- 查看日志。
- 进入 app 容器。
- 进入 MySQL 容器。
- 停止服务。

## 检查标准

如果你做完了，至少应该能清楚解释：

- Compose 和单个 `docker run` 的区别。
- `services`、`ports`、`volumes` 的作用。
- 容器间为什么用服务名通信。
- `docker compose down` 和 `docker compose down -v` 的区别。
- 为什么数据库容器要挂载 volume。
