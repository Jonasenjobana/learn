# Day1 练习题：GitHub Actions 基础与第一个 CI

这一天专门练 GitHub Actions 的最小 workflow。

## 题目 1：创建 workflow 文件

在一个 GitHub 仓库中创建：

```text
.github/workflows/ci.yml
```

要求：

- 文件名可以叫 `ci.yml`。
- 必须放在 `.github/workflows` 目录下。

## 题目 2：配置触发条件

配置 workflow 在下面场景运行：

- push 到 `main`
- pull request 指向 `main`

要求：

- 使用 `on.push.branches`。
- 使用 `on.pull_request.branches`。

## 题目 3：创建第一个 job

创建一个名为 `check` 的 job。

要求：

- 使用 `ubuntu-latest`。
- 至少包含一个 step。

## 题目 4：使用 checkout

添加：

```yaml
- uses: actions/checkout@v4
```

要求：

- 说明它的作用。
- 尝试去掉它，观察后续读取项目文件时会发生什么。

## 题目 5：执行命令

添加几个 `run` step：

```yaml
- run: pwd
- run: ls
- run: echo "hello ci"
```

要求：

- 在 Actions 日志中找到输出。

## 题目 6：查看上下文信息

输出一些 GitHub 默认变量：

```yaml
- run: |
    echo "$GITHUB_REPOSITORY"
    echo "$GITHUB_REF"
    echo "$GITHUB_SHA"
```

要求：

- 说明这些变量大概代表什么。

## 题目 7：制造一次失败

添加一条错误命令：

```yaml
- run: exit 1
```

要求：

- 观察 workflow 失败状态。
- 查看失败 step 的日志。
- 修复后重新提交。

## 题目 8：多 step 顺序

创建 3 个 step，分别输出：

```text
step 1
step 2
step 3
```

要求：

- 观察它们是否按顺序执行。

## 题目 9：修改触发分支

把触发分支从 `main` 改成你的实际默认分支，例如 `master`。

要求：

- 推送到对应分支。
- 确认 workflow 能触发。

## 题目 10：综合小练习

写一个完整 CI：

- push/PR 触发。
- checkout 代码。
- 输出当前目录。
- 输出文件列表。
- 输出 Node 版本。
- 输出一段自定义文字。

## 检查标准

如果你做完了，至少应该能清楚解释：

- workflow 文件为什么要放在 `.github/workflows`。
- `on`、`jobs`、`steps` 分别是什么。
- `runs-on` 决定什么。
- `actions/checkout` 的作用。
- 如何从 Actions 页面查看日志。
