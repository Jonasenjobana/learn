# Day2 练习题：Node 项目 CI、缓存与构建产物

这一天专门练真实 Node/前端项目的 CI。

## 题目 1：确认项目脚本

打开项目的 `package.json`，确认是否有：

- `lint`
- `test`
- `build`

要求：

- 有就写进 workflow。
- 没有就先不要硬写，避免 CI 必然失败。

## 题目 2：配置 Node 版本

使用 `actions/setup-node@v4` 配置 Node。

要求：

- 指定项目实际 Node 版本。
- 打印 `node -v` 和 `npm -v`。

## 题目 3：使用 npm ci

在 CI 中安装依赖：

```yaml
- run: npm ci
```

要求：

- 说明它和 `npm install` 的区别。
- 确认项目有 `package-lock.json`。

## 题目 4：开启 npm 缓存

配置：

```yaml
cache: npm
```

要求：

- 观察第一次和第二次运行时依赖安装速度。
- 在日志中寻找 cache 相关输出。

## 题目 5：运行 lint

添加：

```yaml
- run: npm run lint
```

要求：

- 项目必须有 `lint` 脚本。
- 故意制造一个 lint 错误并观察失败。

## 题目 6：运行测试

添加：

```yaml
- run: npm test
```

要求：

- 项目必须有测试脚本。
- 观察测试失败时 workflow 状态。

## 题目 7：运行构建

添加：

```yaml
- run: npm run build
```

要求：

- 确认构建输出目录。
- 前端项目常见是 `dist`。

## 题目 8：上传构建产物

使用 `actions/upload-artifact@v4` 上传 `dist`。

要求：

- artifact 名称叫 `dist`。
- 在 Actions 页面下载查看。

## 题目 9：Node 版本矩阵

配置多个 Node 版本：

```yaml
node-version: [20, 22]
```

要求：

- 观察 workflow 是否跑了多次 job。
- 说明矩阵适合什么场景。

## 题目 10：综合小练习

为一个 Node 或前端项目配置 CI：

- checkout
- setup-node
- npm ci
- lint
- test
- build
- upload artifact

## 检查标准

如果你做完了，至少应该能清楚解释：

- 为什么 CI 里推荐 `npm ci`。
- `actions/setup-node` 的作用。
- npm 缓存能优化什么。
- artifacts 适合保存什么。
- 为什么 CI 失败时不应该继续部署。
