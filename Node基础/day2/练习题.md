# Day2 练习题：路径、文件系统与命令行脚本

这一天主要练 Node 操作本地文件和目录的能力。

## 题目 1：路径拼接

使用 `path.join()` 拼接下面路径：

```text
data
users.json
```

要求：

- 输出拼接结果
- 不要手动写 `/` 或 `\`

## 题目 2：路径信息

给定路径：

```text
C:\study\node\data\users.json
```

输出：

- 文件名
- 目录名
- 扩展名

要求：

- 使用 `path.basename()`
- 使用 `path.dirname()`
- 使用 `path.extname()`

## 题目 3：写入学习日志

写一个脚本，创建 `study_log.txt`，写入：

```text
今天学习 Node 文件系统
```

要求：

- 使用 `fs/promises`
- 使用 `writeFile()`
- 编码使用 `utf-8`

## 题目 4：读取学习日志

读取题目 3 创建的 `study_log.txt` 并打印内容。

要求：

- 使用 `readFile()`
- 使用 `async / await`
- 使用 `try / catch`

## 题目 5：追加日志

给 `study_log.txt` 追加一行：

```text
明天继续学习 HTTP 服务
```

要求：

- 使用 `appendFile()`
- 注意换行

## 题目 6：命令行读取文件

写一个脚本：

```bash
node read-file.js study_log.txt
```

要求：

- 从 `process.argv` 读取文件名
- 如果没有传文件名，打印提示
- 如果文件不存在，打印错误信息

## 题目 7：复制文件

写一个脚本：

```bash
node copy.js source.txt target.txt
```

要求：

- 接收两个命令行参数
- 使用 `copyFile()`
- 参数不足时提示用户

## 题目 8：统计文本

读取一个文本文件，统计：

- 总字符数
- 总行数
- 非空行数

要求：

- 使用 `split()`
- 空行不计入非空行

## 题目 9：扫描目录

写一个脚本：

```bash
node list.js .
```

输出当前目录下每个文件的信息：

- 文件名
- 扩展名
- 文件大小

要求：

- 使用 `readdir()`
- 使用 `stat()`
- 使用 `path.extname()`

## 题目 10：JSON 格式化

读取一个 JSON 文件，把它格式化写入另一个文件。

示例：

```bash
node format-json.js user.json user.pretty.json
```

要求：

- 使用 `JSON.parse()`
- 使用 `JSON.stringify(data, null, 2)`
- JSON 不合法时要打印错误原因

## 题目 11：综合小练习

写一个命令行工具，根据第一个参数决定操作：

```bash
node tool.js read note.txt
node tool.js append note.txt "hello"
node tool.js info note.txt
```

要求：

- `read`：读取文件
- `append`：追加内容
- `info`：输出文件大小和扩展名
- 无效命令要给出提示

## 检查标准

如果你做完了，至少应该能清楚解释：

- 为什么路径处理要用 `path`。
- `writeFile()` 和 `appendFile()` 的区别。
- 为什么文件读写通常要写成异步。
- `process.argv` 的业务参数从哪里开始。
- 读取文件失败时为什么要做错误处理。
