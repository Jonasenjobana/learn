# Day1 练习题：Node 环境、npm 与模块系统

这一天主要练 Node 的运行方式和项目基础，不重复练 JavaScript 的基础语法。

## 题目 1：确认环境

在命令行分别执行：

```bash
node -v
npm -v
```

要求：

- 记录输出的版本号
- 能说清楚 `node` 和 `npm` 分别是什么

## 题目 2：第一个 Node 程序

新建一个 JS 文件，输出：

```text
Hello Node
```

要求：

- 使用 `console.log`
- 使用 `node 文件名` 运行

## 题目 3：打印运行环境

写一个程序，输出：

- 当前命令执行目录
- 当前操作系统平台
- 当前 Node 进程参数

提示：

```js
process.cwd()
process.platform
process.argv
```

## 题目 4：命令行参数

写一个程序，接收一个名字参数，输出：

```text
你好，Alice
```

运行示例：

```bash
node hello.js Alice
```

要求：

- 使用 `process.argv`
- 如果没有传名字，默认输出 `你好，Node`

## 题目 5：创建 package.json

在一个新目录里执行：

```bash
npm init -y
```

要求：

- 打开生成的 `package.json`
- 找到 `name`、`version`、`scripts`
- 尝试添加一个 `start` 脚本

## 题目 6：npm scripts

在 `package.json` 里添加：

```json
{
  "scripts": {
    "start": "node index.js"
  }
}
```

要求：

- 使用 `npm run start` 运行
- 理解它和 `node index.js` 的关系

## 题目 7：CommonJS 导出函数

新建两个文件：

```text
math.js
index.js
```

在 `math.js` 里导出一个加法函数，在 `index.js` 里导入并调用。

要求：

- 使用 `module.exports`
- 使用 `require`

## 题目 8：ESM 基础认识

尝试创建一个带有下面配置的 `package.json`：

```json
{
  "type": "module"
}
```

然后写：

```js
export function add(a, b) {
  return a + b
}
```

要求：

- 使用 `import` 导入函数
- 观察它和 CommonJS 的写法差异

## 题目 9：依赖分类

不用真的安装也可以，先解释下面两条命令的区别：

```bash
npm install lodash
npm install eslint -D
```

要求：

- 说明哪个会进入 `dependencies`
- 说明哪个会进入 `devDependencies`

## 题目 10：综合小练习

写一个命令行小程序：

- 接收两个数字参数
- 输出它们的和
- 如果参数不足，输出提示

运行示例：

```bash
node add.js 3 5
```

输出：

```text
8
```

## 检查标准

如果你做完了，至少应该能清楚解释：

- Node 和浏览器 JavaScript 的主要区别。
- `package.json` 的作用。
- `npm run start` 为什么能运行脚本。
- CommonJS 和 ESM 的基本写法差异。
- `process.argv` 里为什么前两个值通常不是业务参数。
